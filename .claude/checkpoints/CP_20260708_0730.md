## Key Decisions

- **OAuth invece di Service Account**: il worker riutilizza `GSC_CLIENT_ID/SECRET/REFRESH_TOKEN` già configurato per GSC — stesso token funziona per tutte le API Google (GSC, GA4, PageSpeed). Nessun setup GCP nuovo richiesto.
- **Cache 1 ora in KV** (`seo:data`): il bottone "↻ Aggiorna" forza refresh, cron aggiorna ogni ora via `syncGscDaily` esistente
- **sitemap/submit usa PUT**: la GSC API `sitemaps/{feedpath}` richiede PUT, non POST
- **GA4 graceful degradation**: se `GA4_PROPERTY_ID` non è impostato, la dashboard mostra "GA4 non configurato" — tutto il resto funziona

## Failed Approaches

- **Service Account GCP**: inizialmente pianificato come setup, scoperto che il worker usa già OAuth GSC — Service Account non necessario, semplifica il setup
- **GSC OAuth scope**: GA4 richiede `analytics.readonly` separato dallo scope `webmasters` — scoperta tramite ricerca API

## Do NOT

- Non tentare altri upload Edge finché la submission in coda non è chiusa (API blocca con `InProgressSubmission`)
- Non forzare sitemap submit con POST (API rifiuta con errore 400)

## Failed Approaches

- **QEMU Android emulator**: fallisce "CPU Architecture arm is not supported by QEMU2" — no KVM
- **Waydroid**: richiede `CONFIG_ANDROID_BINDER_IPC` nel kernel — 6.18.31 non ce l'ha
- **CF Pages hosting APK**: limite 25MB hardcoded, APK 46MB
- **USB debugging stabile**: cavo si riconnette in MTP mode — workaround: ADB over WiFi

## Do NOT
- Non buildare APK senza `JAVA_HOME=~/jdk17/jdk-17.0.12+7`
- Non pubblicare su Play Store per ora
- Non promettere video ad blocking su mobile (DNS limitato)
- Non deployare worker con creds .env (D1-only)
- Non tracciare DNS queries

## Key Decisions

- **APK su GitHub Releases**: nessun limite size, canonico open-source
- **Licenza mobile**: ECDSA P-256 verificata lato server (mobile non ha crypto.subtle)
- **APK firmato con keystore autofirmato**: per sideloading, non Google Play Sign
- **ADB WiFi**: paired su `192.168.1.213:45317` → `adb-RFCX115AL9J` (S24 Ultra)

## Architecture

```
worker.js                          ← GET /verify-mobile-license (ECDSA P-256)
AdOff/lib/services/
  license_service.dart            ← UUID deviceId, POST /trial, GET verify-mobile-license
  vpn_service.dart               ← MethodChannel → Kotlin VpnService
AdOff/android/app/src/main/kotlin/app/adoff/
  VpnService.kt                  ← DNS interceptor IPv4/UDP, blocklist, NXDOMAIN
  MainActivity.kt                ← MethodChannel handler
site/
  android.html                   ← Landing page APK download (link → GitHub Releases)
sviluppo/
  adoff-android-README.md        ← README per repo eroslifestyle/adoff-android
  license-system/worker.js       ← worker aggiornato
```

## Resume Instructions

### Worker deploy (richiede OAuth)
```bash
cd sviluppo/license-system
wrangler login   # OAuth → Workers edit permission
wrangler deploy worker.js --name adoff-license
```

### F-Droid
```bash

# Aggiungere metadata/app.adoff.yaml con package name, version, repo URL
git push origin master
# Submit MR a gitlab.com/fdroid/fdroiddata
```

### GitHub repo eroslifestyle/adoff-android

mkdir adoff-android && cd adoff-android

### Firebase Console
1. console.firebase.google.com → nuovo progetto "AdOff"
2. Aggiungere Android app: package `app.adoff`
3. Download `google-services.json` → mettere in `AdOff/android/app/`
4. Aggiungere deps: `firebase_crashlytics`, `firebase_core`
5. Ridisegnare APK + push

### Rebuild APK (se servono modifiche)
```bash
export JAVA_HOME=~/jdk17/jdk-17.0.12+7
export ANDROID_HOME=~/android-sdk
export PATH=$JAVA_HOME/bin:$PATH:$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools
cd AdOff
flutter clean && flutter pub get && flutter build apk --release
# Firmare
source ~/.secrets/adoff-android-keystore.env
$ANDROID_HOME/build-tools/36.0.0/apksigner sign \
  --ks ~/.secrets/adoff-android-release.jks \
  --ks-pass "pass:$STORE_PASS" \
  --key-pass "pass:$KEY_PASS" \
  --ks-key-alias adoff-android \
  build/app/outputs/flutter-apk/app-release.apk
```

### Installare APK su telefono (WiFi ADB)
```bash
ADB=~/android-sdk/platform-tools/adb
sudo $ADB connect 192.168.1.213:45317
sudo $ADB install -r AdOff/build/app/outputs/flutter-apk/app-release.apk
sudo $ADB shell am start -n app.adoff/.MainActivity
```

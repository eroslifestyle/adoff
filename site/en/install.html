🧭 Before you start Safari is more restrictive than other browsers. It requires Xcode (free Apple program) to install extensions. The process takes about 30 minutes the first time. You need macOS Ventura 13 or later, Safari 16.4 or later. Don't worry if you've never opened Terminal, I'll guide you step by step.
1
Extract the ZIP file
1.1 Open Finder (the smiling icon in the Dock at the bottom).
1.2 In the left sidebar, click Downloads.
1.3 Find the file adoff-safari.zip (the one you just downloaded).
1.4 Double-click the file. macOS will extract it automatically and create a folder adoff-safari next to the ZIP.
1.5 Do not move or rename this folder. Leave it in Downloads.
2
Install Xcode (free, ~10 GB)
2.1 Open Mac App Store (blue icon with white "A", in the Dock or Launchpad).
2.2 In the search bar at the top left, type Xcode and press Return.
2.3 Click Download or Install on the Xcode entry (Apple icon, by Apple).
2.4 Sign in with your Apple ID if prompted (the same one from App Store).
2.5 Wait for the download. Xcode is large (~10 GB), could take 20-60 minutes.
2.6 When ready, open Xcode at least once: click the icon in Launchpad or Applications.
2.7 A window appears with license terms. Click Agree.
2.8 Wait for Xcode to finish initial setup (1-2 minutes), then close it (Cmd+Q).
💡 If your Mac doesn't have enough space, free up at least 15 GB before starting.
3
Open Terminal and run the command
3.1 Press Cmd + Space together. A search bar (Spotlight) opens at the top.
3.2 Type Terminal and press Return. A white/black window opens (Terminal, nothing scary, it's a normal Mac program).
3.3 Copy this command exactly (click the gray box to select it, then Cmd+C):
xcrun safari-web-extension-converter ~/Downloads/adoff-safari
3.4 Click in the Terminal window, paste with Cmd+V, then press Return.
3.5 Terminal asks "Continue?". Type y and press Return.
3.6 Wait 30-60 seconds. Xcode opens automatically with a project "AdOff".
3.7 In Xcode, click the ▶ Play button in the top left (or Cmd+R).
3.8 Compilation: wait 1-3 minutes. A small "AdOff" window might open. You can close it (the extension is already installed).
⚠ If you get "permission denied" or "command not found": open Xcode, Settings, Locations, Command Line Tools and select the latest version. Then retry step 3.4.
4
Enable Development menu in Safari
4.1 Open Safari (compass icon in the Dock).
4.2 In the menu bar at the top, click Safari → Settings (or Cmd+,).
4.3 A window opens with tabs at the top. Click Advanced (the last one, gear icon).
4.4 At the bottom, find the option Show features for web developers (or "Show Develop menu" in older versions).
4.5 Check the box. A new Develop menu appears in Safari's menu bar.
4.6 Close the Settings window.
5
Allow unsigned extensions
5.1 In Safari's menu bar, click the new Develop menu.
5.2 Scroll the menu and click Allow Unsigned Extensions.
5.3 Safari asks for your Mac password (the one you use to log in). Type it and press Return.
5.4 The option is now active (you can verify: reopening the Develop menu, "Allow Unsigned Extensions" has a checkmark).
⚠ Important: this option disables automatically every time you restart Safari. You need to re-enable it (steps 5.1-5.3) each system restart.
6
Enable AdOff
6.1 Reopen Safari Settings: Safari → Settings (or Cmd+,).
6.2 Click the Extensions tab (puzzle icon).
6.3 In the left list, you should see AdOff. Check the box next to it.
6.4 A popup appears asking for site access permissions. Click Allow on All Websites (or "Allow on all websites").
6.5 Close Settings. Visit any site with ads (news, video platform, forum). Ads are blocked.
6.6 The AdOff icon (an "A" with red overlay) appears automatically in Safari's toolbar in the top right. Click it to open the popup with controls.
✓ Done. AdOff is active. You have 15 days of free trial trial free. The icon shows the countdown.
🔒 Make installation permanent Unsigned extensions disable when Safari restarts. To use AdOff permanently without repeating step 5 every time, you have two options: (1) Subscribe to Apple Developer program ($99/year) and sign the extension yourself, or (2) Wait for Mac App Store release. Frequently Asked Questions

Do I need to keep the folder on my computer? ▼
Yes. The browser reads the extension files from the folder. If you delete it, the extension stops working. Put it somewhere safe (for example, Documents).
Is it safe to enable Developer Mode? ▼
Yes, completely safe. Developer Mode simply allows you to install extensions from a folder instead of the store. It doesn't change anything else in the browser.
The browser says "Disable developer extensions" - what do I do? ▼
Some browsers show a warning on startup when you have manually installed extensions. Just close the warning. The extension continues to work normally. When AdOff is on the official store, this warning will disappear.
How do I update AdOff? ▼
Download the new version from the site, extract the ZIP to the same folder (overwrite the files), then go to your browser's extensions page and click the "Reload" arrow on AdOff.
Does it work on all sites? ▼
Yes. AdOff blocks ads on all websites (banners, popups, trackers, social media, news, blogs). Video ad blocking on streaming platforms is available in Pro/Trial. You can also pause blocking on specific sites if needed.
How do I uninstall AdOff? ▼
Go to your browser's extensions page, find AdOff and click "Remove". Then you can delete the folder from your computer.
Which browsers are supported? ▼
AdOff supports all Chromium browsers (Chrome, Edge, Brave, Opera, Vivaldi, Arc) with 100% feature compatibility, Firefox with 95%, and Safari 16.4+ on macOS Ventura+ with 95% (requires Xcode). On Android use Kiwi Browser or Firefox. iOS support coming soon.
Does it work on mobile? ▼
On Android, install Kiwi Browser (free from Play Store) which supports Chrome extensions. On iOS, browser extensions aren't yet supported. We're working on it.

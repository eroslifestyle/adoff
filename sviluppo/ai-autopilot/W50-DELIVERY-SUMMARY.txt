================================================================================
TROUBLESHOOTING
================================================================================

Q: "FLUX timeout" error in media_queue
A: cyber-purge bridge is slow or unreachable. Check:
   - http://host.docker.internal:8792 accessible from n8n container
   - Increase timeout or reduce language variants
   - Check host.docker.internal is valid in your Docker setup

Q: "image-gen enqueue failed"
A: W98 webhook unreachable. Check:
   - W98 is running and active
   - /webhook/image-gen endpoint exists in W98
   - n8n container has network access to localhost:5678

Q: "media_queue is empty after schedule trigger"
A: No approved drafts or all have media_queue entries. Check:
   - SELECT * FROM gemini_copy_drafts WHERE status='approved' AND asset_type='caption_social'
   - Clean up old media_queue rows if testing: TRUNCATE adoff_autopilot.media_queue CASCADE
   - Verify schedule trigger is active (clock icon in n8n)

Q: HMAC validation fails on webhook
A: Secret mismatch. Check:
   - WEBHOOK_HMAC_SECRET environment variable is set
   - Calculate expected HMAC: echo -n '<body>' | openssl dgst -sha256 -mac HMAC -macopt key:<secret>
   - Signature header is X-AdOff-Signature (case-sensitive)
